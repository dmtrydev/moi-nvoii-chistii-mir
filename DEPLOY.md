# Деплой на Render.com

Один сервис на Render отдаёт и сайт (фронт), и API (бэкенд). Заказчику даёте одну ссылку вида `https://ваш-сервис.onrender.com`.

---

## Что нужно заранее

1. **Аккаунт на [Render.com](https://render.com)** — регистрация бесплатная (достаточно email или вход через GitHub).
2. **Проект в репозитории на GitHub** (или GitLab / Bitbucket). Код проекта должен быть запушен в репозиторий — Render подключается к нему и собирает приложение из кода.

Если репозитория ещё нет:
- Создайте репозиторий на [GitHub](https://github.com).
- В папке проекта выполните:
  ```bash
  git init
  git add .
  git commit -m "Initial commit"
  git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
  git push -u origin main
  ```
  (подставьте свой логин и имя репозитория).

---

## Пошаговая инструкция

### Шаг 1. Войти в Render

1. Откройте [https://dashboard.render.com](https://dashboard.render.com).
2. Войдите в аккаунт (Sign In) или зарегистрируйтесь (Get Started).
3. Рекомендуется привязать GitHub: **Account Settings** → **Connect GitHub** — так Render получит доступ к вашим репозиториям.

---

### Шаг 2. Создать новый Web Service

1. На главной странице Dashboard нажмите **New** (синяя кнопка).
2. В списке типов сервисов выберите **Web Service**.
3. В блоке **Connect a repository** найдите и выберите репозиторий с этим проектом. Если репозитория нет в списке — нажмите **Configure account** и выдайте Render доступ к нужному репо на GitHub.
4. После выбора репозитория нажмите **Connect** (или **Next**).

---

### Шаг 3. Настроить сервис

Заполните форму создания Web Service по пунктам.

#### Имя (Name)

- Например: `zakaz2-demo` или `ecoplatform`.
- По нему будет сформирован URL: `https://zakaz2-demo.onrender.com`.

#### Регион (Region)

- Оставьте по умолчанию (например, Frankfurt) или выберите ближайший к заказчику.

#### Ветка (Branch)

- Укажите ветку, из которой деплоить (обычно `main` или `master`).

#### Root Directory

- Оставьте **пустым**. Проект должен собираться из корня репозитория (там лежат `package.json`, папка `server`, папка `src` и т.д.).

#### Runtime

- Выберите **Node**.

#### Build Command (команда сборки)

Скопируйте и вставьте **целиком**:

```bash
npm install && cd server && npm install && cd .. && npm run build:client
```

Что делает команда:
- `npm install` — ставит зависимости корня проекта (фронт).
- `cd server && npm install` — ставит зависимости сервера (Express, pdf-parse и т.д.).
- `cd .. && npm run build:client` — возврат в корень и сборка фронта в папку `dist`.

#### Start Command (команда запуска)

Введите:

```bash
npm start
```

Эта команда запускает `node server/index.js`: поднимается один процесс, который отдаёт и API (`/api/...`), и статику сайта из `dist`.

#### Instance Type

- Для демо заказчику достаточно **Free**. После неактивности сервис «засыпает», первый запрос может открываться 30–60 секунд — это нормально для бесплатного плана.

---

### Шаг 4. Добавить переменные окружения (Environment Variables)

Без этих переменных анализ лицензий через Timeweb работать не будет.

1. В той же форме создания сервиса найдите блок **Environment Variables** (или **Environment**).
2. Нажмите **Add Environment Variable**.
3. Добавьте две переменные по одной:

| Key (имя)              | Value (значение) |
|------------------------|------------------|
| `TIMEWEB_ACCESS_ID`    | `adea83eb-75b9-41cb-95e0-6cb244587767` |
| `TIMEWEB_BEARER_TOKEN` | Ваш JWT-токен из личного кабинета Timeweb (длинная строка, начинается с `eyJ...`) |

- Имя вводите **точно** как в таблице (без пробелов, с подчёркиваниями).
- Значение для `TIMEWEB_BEARER_TOKEN` вставьте целиком, без кавычек и пробелов в начале/конце.

**Важно:** переменную `VITE_API_URL` на Render **не** добавляйте — на деплое фронт должен ходить на тот же хост (`/api/...`), иначе запросы пойдут не туда.

---

### Шаг 5. Создать сервис и дождаться деплоя

1. Нажмите **Create Web Service** (внизу формы).
2. Render начнёт первый деплой: клонирует репозиторий, выполнит Build Command, затем Start Command.
3. В верхней части страницы сервиса отображается лог (Build Log, затем Deploy Log). Дождитесь сообщения вроде **Your service is live at …**.
4. Ссылка на сайт будет вида:  
   `https://zakaz2-demo.onrender.com`  
   (имя подставлено то, что вы указали в **Name**). Эту ссылку и нужно отдать заказчику.

---

### Шаг 6. Проверить работу

1. Откройте в браузере ссылку сервиса — должна открыться главная страница («Умная экоплатформа»).
2. Перейдите на страницу загрузки (кнопка «Разместить объект» или URL `…/upload`).
3. Загрузите тестовый PDF с текстом лицензии — должен пройти этап «Нейросеть анализирует лицензию…» и появиться форма с данными.

Если загрузка выдаёт ошибку — проверьте в **Environment** на Render, что переменные `TIMEWEB_ACCESS_ID` и `TIMEWEB_BEARER_TOKEN` заданы без опечаток.

---

## Как обновить сайт после изменений в коде

1. Закоммитьте и запушьте изменения в тот же репозиторий и ветку, которые подключены к Render:
   ```bash
   git add .
   git commit -m "Описание изменений"
   git push origin main
   ```
2. В [Dashboard Render](https://dashboard.render.com) откройте ваш Web Service.
3. Render обычно запускает новый деплой автоматически после `git push`. Если нет — нажмите **Manual Deploy** → **Deploy latest commit**.
4. Дождитесь окончания деплоя — заказчик снова увидит обновлённую версию по той же ссылке.

---

## Важные замечания

- **Одна ссылка:** и сайт, и API работают с одного домена. Фронт обращается к `/api/analyze-license` и т.п. — дополнительно ничего настраивать не нужно.
- **Free-план:** сервис может «засыпать» после 15 минут неактивности. Первый запрос после этого может выполняться 30–60 секунд — предупредите заказчика.
- **Секреты:** токен Timeweb храните только в Environment Variables на Render и в своём локальном `.env`. В репозиторий `.env` не коммитьте (он уже в `.gitignore`).

## Локальная проверка перед первым деплоем (по желанию)

Чтобы убедиться, что сборка и запуск у вас проходят так же, как на Render:

В **корне** проекта выполните:

```bash
npm install
cd server && npm install && cd ..
npm run build:client
npm start
```

Откройте в браузере [http://localhost:3001](http://localhost:3001). Должны открыться главная, карта и страница загрузки; загрузка PDF будет идти через тот же порт (при настроенном локально `TIMEWEB_ACCESS_ID` и `TIMEWEB_BEARER_TOKEN` в `server/.env`).

После проверки остановите сервер (Ctrl+C) и деплойте на Render по инструкции выше.
